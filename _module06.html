DEFINING AND USING CLASSES    
    

        • Classes define:        
        • What an object is
        • What an object can do
        • Class names start with an uppercase letter
        • Classes organize code into well-categorized areas
        • Classes "carry around" their class's code
        • Classes allow complex behaviors using simple statements
        • Classes correspond to real-world objects
        

        • Declaring a class:
        

        class Animal
            def make_noise
                "Moo!"            
            end            
        end

        >> animal = Animal.new 
        >> animal.make_noise 
        => "Moo!"
        
 
        

        • Command to uninitialize a class:
        >> Object.send(:remove_const, :Animal)
    

    CLASS INSTANCES    
    

        • A class instance is an object created from a class 
    

    CLASS INSTANCE ATTRIBUTES    
    

        • A class instance attribute is a value that persists inside of an instance
        • An instance variable (@variable) is used to store the attributes inside an instance
        

        • Instance variable in a class:
        

        class Animal
            def set_noise(noise)
                @noise = noise            
            end 
            def get_noise
                @noise            
            end            
        end

        >> animal = Animal.new 
        >> animal.set_noise("Quack!") 
        >> animal.make_noise 
        => "Quack!"
        
 
    

    READER/WRITER METHODS    
    

        • Also known as getter/setter methods in other languages
        

        • Refactoring the class with reader/writer methods:
        

        class Animal
            def noise=(noise)
                @noise = noise            
            end 
            def noise
                @noise            
            end            
        end

        >> animal = Animal.new 
        >> animal.noise ="Quack!" 
        >> animal.noise 
        => "Quack!"
        
 
    

    ATTRIBUTE METHODS    
    

        • attr_reader, attr_writer, attr_accessor
        

        • Refactoring the class with attr_accessor method:
        

        class Animal
            attr_accessor :noise            
        end

        >> animal = Animal.new 
        >> animal.noise ="Quack!" 
        >> animal.noise 
        => "Quack!"
        
 
    

    INITIALIZE METHOD    
    

        • Initialize method can contain default values and default methods that run immediately
        

        • Refactoring the class with initialize method:
        

        class Animal
            attr_accessor :noise   
            def initialize(noise)
                puts "A new animal has been instantiated."
                @noise = noise
            end 
        end

        >> animal = Animal.new("Meow!") 
        >> animal.noise 
        => "Meow!"
        
 
    

    CLASS METHODS    
    

        • Class method is a method than can be called on a class, even without an instance of the class
        • To create a class method, a method needs the keyword 'self.' prefixing the method name
        

        • Refactoring the class with a class method:
        

        class Animal
            attr_accessor :noise   
            def self.all_species
                ['cat', 'cow', 'dog', 'duck', 'horse', 'pig']
            end
            def initialize(noise)
                puts "A new animal has been instantiated."
                @noise = noise
            end 
        end

        >> Animal.all_species 
        => ["cat", "cow", "dog", "duck", "horse", "pig"]
        
 
    

    CLASS ATTRIBUTES    
    

        • Class attributes store values that apply to the class generally
        • A class attribute needs the '@@'prefix to the name
        

        • Refactoring the class with a class attribute:
        

        class Animal
            attr_accessor :noise  
            @@species = ['cat', 'cow', 'dog', 'duck', 'horse', 'pig'] 
            def self.all_species
                @@species
            end
            def initialize(noise)
                puts "A new animal has been instantiated."
                @noise = noise
            end 
        end

        >> Animal.all_species 
        => ["cat", "cow", "dog", "duck", "horse", "pig"]
        
 
    

    CLASS READER/WRITER METHODS    
    

        • Class reader/writer methods have to be created
        

        • Refactoring the class with class reader/writer methods:
        

        class Animal
            attr_accessor :noise  
            @@species = ['cat', 'cow', 'dog', 'duck', 'horse', 'pig'] 
            def self.species
                @@species
            end
            def self.species=(array=[])
                @@species = array
            end
            def initialize(noise)
                puts "A new animal has been instantiated."
                @noise = noise
            end 
        end

        >> Animal.species = ['frog', 'fish'] 
        => ["frog", "fish"]
        
 
    

    CLASS INHERITANCE    
    

        • Class inheritance is the bestowal of methods and attributes of another class
        • In Ruby, the parent class is called a superclass, and the child class is called a subclass
        • In Ruby, only one class can be inherited
        

        • Adding a subclass:
        

        class Animal
            attr_accessor :noise  
            @@species = ['cat', 'cow', 'dog', 'duck', 'horse', 'pig'] 
            def self.species
                @@species
            end
            def self.species=(array=[])
                @@species = array
            end
            def initialize(noise)
                puts "A new animal has been instantiated."
                @noise = noise            
            end 
        end

        class Cow < Animal
        end

        >> Clarabelle = Cow.new("Moo!") 
        >> Clarabelle.noise
        => "Moo!"
        
 
    

    SUBCLASS OVERRIDING    
    

        • Class inheritance is the bestowal of methods and attributes of another class
        • In Ruby, the parent class is called a superclass, and the child class is called a subclass
        • In Ruby, only one class can be inherited
        

        • Overriding with a subclass:
        

        class Animal
            attr_accessor :noise, :color  
            @@species = ['cat', 'cow', 'dog', 'duck', 'horse', 'pig'] 
            def self.species
                @@species
            end
            def self.species=(array=[])
                @@species = array
            end
            def initialize(noise, color)
                puts "A new animal has been instantiated."
                @noise = noise
                @color = color
            end 
        end

        class Cow < Animal
            def color
                "The cow's color is #{@color}."
            end
        end

        >> Clarabelle = Cow.new("Moo!") 
        >> Clarabelle.noise   => "Moo!"
        >> Clarabelle.color   => "The cow's color is yellow."
        
 
    

    ACCESSING THE SUPERCLASS    
    

        • Access the method in the superclass by using keyword 'super'
        

        • Overriding with a subclass:
        

        class Animal
            attr_accessor :noise, :color  
            @@species = ['cat', 'cow', 'dog', 'duck', 'horse', 'pig'] 
            def self.species
                @@species
            end
            def self.species=(array=[])
                @@species = array
            end
            def initialize(noise, color)
                puts "A new animal has been instantiated."
                @noise = noise
                @color = color
            end 
        end

        class Cow < Animal
            def color
                "The cow's color is #{@color}."
            end
        end

        class Pig < Animal
            def noise
                super
            end
        end

        >> Wilbur = Pig.new("Oink!","pink") 
        >> Wilbur.noise   => "Oink!"
        >> Wilbur.color   => "pink"
